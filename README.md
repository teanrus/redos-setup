# redos-setup

Скрипт для автоматизации первичной настройки рабочих мест на РЕД ОС. Проект выполняет выборочную установку системных и прикладных компонентов, определяет версию ОС, запрашивает подтверждение пользователя перед каждым дополнительным шагом и загружает артефакты только по мере необходимости.

## Overview

Проект предназначен для сценариев, где требуется быстро подготовить рабочее место на РЕД ОС с повторяемой последовательностью действий:

- обновить систему и базовые репозитории;
- установить стандартный набор прикладного ПО;
- предложить дополнительные компоненты по выбору пользователя;
- выполнить отдельные системные настройки;
- не загружать лишние файлы, если пользователь не подтвердил установку соответствующего компонента.

Скрипт использует модель ленивой загрузки: файл скачивается из GitHub Releases только в момент установки и удаляется после завершения шага.

## Features

- Определение версии ОС через `/etc/os-release`
- Поддержка выборочной установки компонентов
- Загрузка артефактов только по подтверждённым шагам
- Автоматическая очистка временных файлов из `/home/inst`
- Установка пакетов через `dnf` и vendor-архивы из GitHub Releases
- Проверка совместимости `КриптоПро` и `ViPNet` с установленной версией РЕД ОС

## Compatibility

Поддержка версий РЕД ОС в текущем релизе:

| Версия ОС | Статус | Примечание |
| :-------- | :----- | :--------- |
| РЕД ОС 7.x | Полная поддержка | Доступен полный сценарий установки, включая `КриптоПро` и `ViPNet` |
| РЕД ОС 8+ | Частичная поддержка | `КриптоПро` и `ViPNet` из текущего релиза автоматически исключаются как несовместимые |
| Другая Linux ОС | Не тестировалось | Скрипт предупредит, что ОС не распознана как РЕД ОС |

При запуске скрипт:

- определяет ОС и её основную версию;
- выводит информацию о найденной платформе;
- автоматически скрывает несовместимые шаги установки;
- продолжает предлагать остальные компоненты, если они не завязаны на ограниченные пакеты.

## Quick Start

Запуск напрямую из последнего релиза:

```bash
curl -sL https://github.com/teanrus/redos-setup/releases/latest/download/setup.sh | sudo bash
```

Запуск с предварительным сохранением файла:

```bash
wget https://github.com/teanrus/redos-setup/releases/latest/download/setup.sh
chmod +x setup.sh
sudo ./setup.sh
```

Если `wget` отсутствует:

```bash
curl -L -o setup.sh https://github.com/teanrus/redos-setup/releases/latest/download/setup.sh
chmod +x setup.sh
sudo ./setup.sh
```

## Requirements

| Параметр | Значение |
| :------- | :------- |
| ОС | РЕД ОС 7.x или 8+ |
| Права | `root` / `sudo` |
| Интернет | Требуется для загрузки файлов из GitHub Releases |
| RAM | Минимум 2 GB, рекомендуется 4 GB |
| Диск | Не менее 5 GB свободного места |
| Архитектура | `x86_64` |

Совместимость компонентов:

- `РЕД ОС 7.x`: полный сценарий установки
- `РЕД ОС 8+`: без установки `КриптоПро` и `ViPNet` из текущего набора пакетов

## How It Works

Сценарий работы одинаков для базовых и дополнительных компонентов:

1. Скрипт проверяет запуск от `root`.
2. Определяет версию ОС и выводит информацию о совместимости.
3. Создаёт рабочую директорию `/home/inst`.
4. Устанавливает недостающие системные утилиты, если они требуются для дальнейшей работы.
5. Запрашивает подтверждение перед установкой базового набора.
6. Поочерёдно предлагает дополнительные компоненты.
7. Скачивает только подтверждённые артефакты.
8. Удаляет временные файлы после завершения шага.
9. Предлагает системные настройки и перезагрузку.

Пример выбора `ViPNet` на совместимой системе:

```text
Установить ViPNet? (y/n): y
=== Выбор версии ViPNet ===
1. ViPNet Client (без деловой почты)
2. ViPNet + Деловая почта (DP)
Выберите вариант (1 или 2): 1
Загрузка vipnetclient-gui_gost_ru_x86-64_4.15.0-26717.rpm...
✓ Установка ViPNet Client успешно выполнено
```

Пример поведения на РЕД ОС 8+:

```text
=== Начало настройки РЕД ОС ===
Обнаружена ОС: РЕД ОС 8.0
Основная версия РЕД ОС: 8
Внимание: скрипт изначально разрабатывался и тестировался для РЕД ОС 7.3.
Для РЕД ОС 8.0 пакеты КриптоПро и ViPNet из этого скрипта не подходят и будут автоматически исключены из установки.
Остальные компоненты будут предложены к установке как обычно.
```

## System Changes

Скрипт вносит изменения в систему. Перед использованием имеет смысл учитывать следующие действия:

- обновляет пакеты через `dnf`;
- может изменить `/etc/selinux/config` и перевести `SELINUX=enforcing` в `SELINUX=disabled`;
- может добавить `max_parallel_downloads=10` в `/etc/dnf/dnf.conf`;
- создаёт рабочую директорию `/home/inst`;
- добавляет репозитории и устанавливает сторонние пакеты;
- может обновить конфигурацию GRUB;
- может включить `fstrim.timer`;
- может добавить `xrandr --output HDMI-3 --primary` в `/etc/gdm/Init/Default`.

### Optional System Configuration

| Настройка | Описание |
| :-------- | :------- |
| TRIM для SSD | Включение `fstrim.timer` |
| Обновление GRUB | Пересоздание `/boot/grub2/grub.cfg` |
| Моноблок KSG | Добавление `xrandr --output HDMI-3 --primary` перед `exit 0` в `/etc/gdm/Init/Default` |

## Component-Specific Notes

Telegram:
- Распаковывается в `/opt/telegram`
- Создаётся символическая ссылка `/usr/bin/telegram`
- Добавляется ярлык в меню приложений

Шрифты Liberation:
- Устанавливаются в `/usr/share/fonts/liberation`
- После установки выполняется `fc-cache -fv`

КриптоПро:
- Сначала устанавливаются зависимые пакеты `ifd-rutokens`, `token-manager`, `gostcryptogui`, `caja-gostcryptogui`
- Затем распаковывается архив `kriptopror4.tar.gz`
- Запускается `install_gui.sh` или альтернативный сценарий установки
- На РЕД ОС 8+ блок автоматически пропускается как несовместимый

ViPNet:
- Client-версия устанавливается из одного RPM-пакета
- DP-версия устанавливается из архива с несколькими RPM-пакетами
- На РЕД ОС 8+ блок автоматически пропускается как несовместимый

## Development

Если вы дорабатываете проект:

- проверяйте изменения в `scripts/setup.sh` на совместимой тестовой системе;
- отдельно учитывайте различия между РЕД ОС `7.x` и `8+`;
- не публикуйте в релизы пакеты, которые не соответствуют целевой версии ОС;
- поддерживайте соответствие между логикой скрипта и документацией в `README.md`.

## Security and Operational Notes

- Скрипт запускается с правами `root`.
- Артефакты загружаются из GitHub Releases этого репозитория.
- Для коммерческих продуктов требуются действующие лицензии.
- Перед запуском рекомендуется резервное копирование важных данных.
- Для РЕД ОС 8+ совместимые дистрибутивы `КриптоПро` и `ViPNet` должны подготавливаться отдельно, так как текущий релиз их не устанавливает.
